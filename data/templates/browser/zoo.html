Localisation

Localisation is supported in Gmod only. Desktop will simply ignore it.
Any element can have a translate attribute with the target localisation key. The text value of that element will be replaced if a match is found.

  • Code: Spawn <span translate="addons.entity">Entities</span> and <span translate="addons.vehicle">Vehicles</span>
  • Example: Spawn Entities and Vehicles

If no match can be found for the user's language (or the user is out-of-game), the existing text value will be kept. For this reason, all strings should be the same as their default English value.

  • Code: This key doesn't exist so will be kept as <span translate="zimbabwe">bazinga</span>
  • Example: This key doesn't exist so will be kept as bazinga

Text case helper

You can adjust the case of text with the helper classes, case, Case, and CASE.

  • case = lowercase everything: eg <span class="case">woRDS ArE hERe</span> = woRDS ArE hERe
  • Case = Capitalise First Letter Of Each Word BUT PRESERVES The REST AS IS: eg <span class="Case">words arE heRE</span> = words arE heRE
  • CASE = UPPERCASE EVERYTHING: eg <span class="CASE">words ArE hERe</span> = words ArE hERe

Localisation

This is particularly useful when used with localisation strings (see the section above) as they may not be the case you want
eg if "addons.entity" returns "Entities" but you want "entities"...

  • Code: these are my favourite <span translate="addons.entity" class="case">EnTiTiEs</span>
  • Result: these are my favourite EnTiTiEs

Input/form elements

HTML <input> elements can be styled for consistent display between desktop browsers and Gmod. Elements adjust for light and dark mode, hover/focus/click, etc.
Use class "form_input" for text inputs and "form_btn" for button inputs. Not required to be used inside an actual form.

  • class="form_input" on type=text -
  • class="form_btn" on type=button/submit and <button> -

Links

Basic link styling

Simple links can have the textlink class for basic styling. This can also be used on javascript links.

  • Code: See the <a class="textlink" href="/news">News</a> page for updates.
  • Example: See the News page for updates.

Hover-state icons

Links can also have a 2-state icon similar to the navigation icons. Add a span with a linkicon class with the background-image for the icon. Styling is handled automatically. Hover link for demo.

  • Code: Find out more about <a class="textlink" href="#"><span class="linkicon" style="background-image: url('/assets/rustmb/combined/unused/banana.png');"></span>bananas</a> here.
  • Example: Find out more about bananas here.

cbalert

cbalert() is an alternative to standard javascript alert() and can be called in the same situations.

  • cbalert(text);
  • cbalert(title, text);

Tooltips

Tooltips can be added to any element by using the tooltip attribute with the tooltip text, and a class to specify which direction.
Element with the tooltip must be position: relative, absolute, or fixed. Elements with the default position: static will not work.
Tooltips can't break out of an overflow: hidden parent.

Example: <span class="tt_topleft" tooltip="I am a tooltip!">Hover me</span>
